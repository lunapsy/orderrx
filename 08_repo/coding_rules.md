# Coding Rules

## 설계 규칙
- 명시적 인터페이스 우선
- schema version 필수
- 민감정보 관련 코드는 테스트 없이 병합 금지
- provider별 분기 로직을 application layer에 두지 말 것
- adapter는 선언형 규칙 우선, 코드 훅은 최소화

## 코드 구조 규칙
- 역할별로 폴더와 파일을 분리한다. 한 파일에 여러 역할을 섞지 않는다.
- 각 파일 상단에는 해당 파일의 역할을 명시하는 헤더 주석을 단다.
- 모든 함수에는 목적, 입력, 출력, 부작용을 설명하는 주석을 단다.
- 공개 인터페이스(public function/class)는 docstring 형식을 사용한다.

## 로깅 규칙
- 각 처리 단계마다 로그를 남겨 에러 위치를 빠르게 추적할 수 있게 한다.
- 로그 레벨은 debug / info / warn / error 로 구분해 사용한다.
- 민감정보(비밀번호, 결제정보, 환자 개인정보, 세션 토큰 등)는 로그에 절대 출력하지 않는다.
- 로그 메시지는 단계명과 컨텍스트(요청 ID, adapter ID 등)를 포함한다.
