# OrderRx Training Track 참여 동의서

- **문서 버전**: 1.0.0
- **최종 수정일**: 2026-07-13
- **운영자**: OrderRx Training Track 운영자 (이하 "운영자")

이 동의서는 OrderRx Training Track Chrome 확장프로그램(이하 "확장")을 설치·사용하는 참여자가,
어떤 데이터가 수집·전송·이용되는지 이해하고 자발적으로 동의했음을 명확히 하기 위한 문서입니다.
확장의 popup 첫 화면에서 이 동의서 요약을 확인하고 "동의합니다"를 누르면 동의가 성립하며,
동의 일시·동의서 버전·확장 버전이 기록됩니다.

---

## 1. 수집 목적

약국 도매 주문 사이트에서의 **업무 흐름(workflow)** — 어떤 순서로 검색하고, 클릭하고,
주문하는지 — 를 학습 데이터로 구조화하여, 약국 주문 업무 자동화 소프트웨어를 개발하기 위함입니다.

## 2. 수집하는 것

참여자가 **직접 등록한 사이트**에서, **수집 활성화 상태일 때만**, 아래 6종 행동 메타데이터를 수집합니다.

| 이벤트 | 내용 |
|---|---|
| page_enter | 페이지 방문 (정제된 URL, 페이지 제목) |
| click | 클릭한 요소의 위치 정보(CSS selector), 종류, 표시 텍스트 앞 40자 |
| field_focus | 입력창의 이름·종류와 입력 **길이** 및 문자 유형 분류 |
| submit | 폼 제출 시 폼 구성 정보 (필드 개수, 필드 이름) |
| navigate | 페이지 이동 (이전/다음 정제 URL) |
| result_rendered | 검색 결과 등이 표시된 시점과 항목 개수 |

## 3. 수집하지 않는 것 (자동 차단)

다음은 **어떤 형태로도 수집·저장·전송되지 않습니다**. 확장이 저장 전 단계에서 자동 차단/마스킹합니다.

- 아이디, **비밀번호, OTP** (비밀번호 필드는 길이조차 기록하지 않음)
- **입력한 값의 원문** (검색어 포함 — 입력창에는 길이와 유형만 기록)
- 카드번호 등 **결제 정보**
- **환자 개인정보**, 주민등록번호
- 전화번호, 이메일 (화면 텍스트에 나타나면 `[PHONE]`, `[EMAIL]`로 치환)
- 세션 토큰, 로그인 쿠키 (URL에서 자동 제거)

## 4. 저장과 전송

1. 수집된 이벤트는 먼저 참여자 브라우저에 저장됩니다.
2. 확장이 주기적으로 운영자가 관리하는 수집 서버(Supabase, 대한민국 서울 리전)에 자동 업로드합니다.
3. 업로드가 확인된 데이터는 브라우저에서 자동 삭제됩니다 (최근 일부는 popup 미리보기용으로 잠시 유지).
4. popup의 "내 데이터 내보내기"로 업로드 전 데이터를 언제든 직접 확인할 수 있습니다.

## 5. 이용 범위와 보호

- 데이터는 **workflow 분석과 자동화 모델 학습**에만 사용합니다.
- 원본(raw) 데이터를 공개 저장소나 제3자에게 제공하지 않습니다.
- 참여자는 실명이 아닌 무작위 ID(participant_id)로만 구분됩니다. 운영자는 이 ID를 실명과
  연결한 목록을 데이터와 분리하여 보관하며, 삭제 요청 처리에만 사용합니다.
- 보관 기간: 프로젝트 종료 또는 참여 철회 시까지.

## 6. 참여자의 권리

참여자는 언제든, 사유 없이, 아래를 할 수 있습니다.

| 권리 | 방법 |
|---|---|
| 일시 정지 | popup의 "수집 활성화" 체크 해제 |
| 특정 사이트만 중단 | popup의 사이트별 체크 해제 또는 삭제 |
| 로컬 데이터 삭제 | popup의 "전체 삭제" 버튼 |
| **참여 철회 + 서버 데이터 삭제** | 운영자에게 participant_id를 알려주고 삭제 요청 — 운영자는 7일 내 해당 ID의 서버 데이터 전부를 삭제하고 결과를 회신 |

## 7. 유의사항

- 일부 주문 사이트의 이용약관에는 자동화 도구·데이터 수집 관련 조항이 있을 수 있습니다.
  본 확장은 사이트에 어떤 요청도 보내지 않고 참여자 본인의 행동 메타데이터만 기록하지만,
  약관 관련 우려가 있으면 해당 사이트를 등록하지 않으면 됩니다.
- 이 확장은 참여자의 브라우저 사용을 대신하지 않으며, 어떤 자동 조작도 수행하지 않습니다.

## 8. 동의의 성립과 기록

- popup 첫 화면에서 이 동의서를 확인하고 **"위 내용을 읽고 이해했으며 동의합니다"** 체크 후
  "동의하고 시작" 버튼을 누르면 동의가 성립합니다.
- 동의 시각, 동의서 버전(1.0.0), 확장 버전, participant_id가 브라우저와 수집 서버에 기록됩니다.
- 동의서 내용이 **의미 있게 변경**되면(수집 항목 확대 등) 버전이 올라가며, 확장이 재동의를 요청합니다.
- 동의하지 않으면 확장은 어떤 데이터도 수집·전송하지 않습니다.

## 9. 문의

수집 내용, 삭제 요청, 기타 문의는 확장을 전달받은 경로(운영자 연락처)로 연락해 주세요.
